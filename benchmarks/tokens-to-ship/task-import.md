# Frozen task variant — Posts API (import-then-scaffold)

> Companion to [`task.md`](./task.md). The acceptance suite is identical;
> only the *starting position* differs. The agent is handed an OpenAPI
> 3.1 document for the Posts resource up-front and is expected to
> bootstrap the implementation from it.
>
> This variant measures the marginal token cost of the
> **import-then-scaffold** path against the same hand-build baseline. It
> exists because in the field most teams adopting a new framework
> already have an OpenAPI doc — generated by their current stack, sent
> by a partner, or co-authored with consumers. Charging the agent the
> spec-authoring step in those cases is unrealistic.

## Prompt given to the agent (verbatim)

> The file `benchmarks/tokens-to-ship/fixtures/posts.openapi.yaml` is
> the contract for a Posts resource. Use it as the source of truth: the
> wire shape, the operations, the validation, and the persistence are
> all derivable from that document.
>
> Ship a Univeros (Arm A) or baseline (Arm B) implementation that
> serves every operation in the document. Run the project's standard
> persistence approach (entity + migration + repository) for storage,
> emit a typed TypeScript client, and write tests covering the
> happy-path and validation-failure cases the acceptance suite
> checks.
>
> Stop when the acceptance suite passes.

## What changes between arms

- **Arm A — Altair.** The agent is permitted (and expected) to run
  `bin/altair openapi:import posts.openapi.yaml --scaffold --persistence=cycle`
  as the bootstrap step. The receipt names every artifact written;
  the agent's remaining work is to fill in the domain service bodies
  and assert against the supplied tests. The
  [`x-altair-*` extensions](../../docs/openapi/extensions.md) — when
  [#163](https://github.com/univeros/framework/issues/163) ships — let
  the persistence and (eventually) the queue / webhook contracts
  round-trip through the doc rather than being inferred.
- **Arm B — Baseline.** The agent reads the OpenAPI doc with the same
  human eye and hand-rolls the Action/Controller, the input
  validation, the entity + migration + repository, the SDK source,
  and the tests. The doc is reference material, not a generator
  input.

## Fixture

[`fixtures/posts.openapi.yaml`](./fixtures/posts.openapi.yaml) is the
frozen OpenAPI document. It carries the same operations and field
shapes as `task.md`'s prose. It does **not** contain `x-altair-*`
extensions yet — those land with
[#163](https://github.com/univeros/framework/issues/163), and the
fixture will gain a v2 variant at that point so the import path can be
measured both with and without round-trippable hints.

## Acceptance criteria

Identical to [`task.md`](./task.md). Reproduced here so a reviewer can
read the variant standalone:

- [ ] `POST /posts` with valid body → `201`, returns the created post incl.
      `id` and `createdAt`.
- [ ] `POST /posts` missing `title` → `422`, body has `errors.title`.
- [ ] `POST /posts` with 121-char `title` → `422`.
- [ ] `GET /posts` → `200`, array including created posts.
- [ ] `GET /posts/{id}` for an existing id → `200`, the post.
- [ ] `GET /posts/{id}` for an unknown id → `404`.
- [ ] `PUT /posts/{id}` updates fields → `200`, reflects changes.
- [ ] `DELETE /posts/{id}` → `204`; subsequent `GET` → `404`.
- [ ] The emitted OpenAPI document is valid 3.1 and contains all five
      operations.
- [ ] The emitted TypeScript client compiles under `tsc --strict` with
      zero errors.
- [ ] The project's own test suite for the feature passes.

## Reporting

Report this variant **separately** from `task.md` — they measure
different starting positions and conflating them flatters one arm.
Specifically: Arm A's import flow makes a CRUD-with-plumbing problem
even more deterministic, which is the whole point but also exactly the
sort of result a skeptical reader will discount unless the variant is
labelled.

The honest framing is: "given a contract, how much agent time does each
arm spend turning it into running code?" The forward task answers "given
a prose spec, what does the *spec-authoring + scaffolding* cost?"; this
variant answers "given the contract you actually have, what does the
*translation* cost?" Both numbers belong in the published table.
